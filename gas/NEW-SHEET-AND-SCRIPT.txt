================================================================================
新しいスプレッドシート ＋ 新しい Apps Script（事故防止用）
店舗用 LINE 公式アカウントは「旧と同じ」にする手順
================================================================================

■ 1. 新しい Google スプレッドシートを作成
  ・名前例: Nail Liberty 予約一覧 v2
  ・シート名を「yoyaku」にする（1枚目をリネーム、または追加して yoyaku を用意）
  ・1行目に必要ならヘッダー行を入れる（既存運用に合わせる）
    ※ sendLineNotification はデータ行を 2 行目から想定（LAST_SENT_ROW 未設定時は startRow=2）

■ 2. スプレッドシートに Apps Script を紐づけて作成
  ・メニュー「拡張機能」→「Apps Script」
  ・既定の Code.gs の中身を消し、同フォルダの nail-reserve-v2.gs の全文を貼り付け
    （または プロジェクト内にファイルを追加して同内容にする）

■ 3. 旧プロジェクトから doGet をコピー
  ・カレンダー用に空き日 JSON を返している「doGet」関数を旧スクリプトからコピー
  ・新プロジェクトの Code.gs 末尾などに貼り付け（doGet は 1 つだけ）

■ 4. 店舗用 LINE を「同じアカウント」にする（重要）
  ・旧 Apps Script を開く → 歯車「プロジェクトの設定」→「スクリプトのプロパティ」
  ・ LINE_ACCESS_TOKEN の値をコピー
  ・新 Apps Script でも「スクリプトのプロパティ」に同じキー名で同じ値を追加
    キー: LINE_ACCESS_TOKEN
    値: （旧と同じ Channel access token）
  → Messaging API のチャネルが同じなので、通知先は従来どおり同じ店舗ボットになります。

■ 5. 新しいスプレッドシート用の LAST_SENT_ROW
  ・新シートではプロパティが別プロジェクトになるため、LAST_SENT_ROW は空の状態から開始されます。
  ・初回の通知は 2 行目以降の未送信行が対象になります（想定どおり）。

■ 6. 認証とデプロイ
  ・エディタで一度実行し、スプレッドシート・外部連携の権限を許可
  ・「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
      実行ユーザー: 自分
      アクセスできるユーザー: 全員（または LIFF から届く構成に合わせる）
  ・発行された「ウェブアプリの URL」をコピー

■ 7. index.html を更新
  ・GAS_URL を 6 でコピーした URL に差し替え
  ・SECRET_TOKEN を nail-reserve-v2.gs の SECRET_TOKEN と同じ文字列にする
    （事故防止のため新トークンに変えてもよい。その場合は .gs と HTML の両方を同じ値に）

■ 8. スプレッドシートの「変更時」トリガーについて（任意）
  ・旧環境で「編集や変更で sendLineNotification」を動かしていた場合:
      新プロジェクトでも同様のトリガーを追加可能です。
  ・nail-reserve-v2.gs では、LIFF からの doPost で行追加した直後にも sendLineNotification() を
    呼んでいるため、API 経由の予約はトリガーなしでも店舗 LINE に届きます。
  ・手入力で行を足したときも通知したい場合のみ、変更時トリガーを追加してください。
  ・doPost 直後とトリガーの二重実行は、LAST_SENT_ROW により通常は二重送信になりません。

■ 9. 旧シート・旧スクリプト
  ・誤接続防止のため、旧 LIFF / 旧 HTML が旧 GAS URL を向いていないか確認してください。
